# Making Mainframes Modern With Forensics
**1970s Financial Code scaled to 2026 Tech**

## The Project
I’m a device hacker who likes to add new features and value to old tech. It is quite fascinating to discover that the global economy runs on "Big Iron" — IBM mainframes that have been clicking and whirring in the background of our everyday reality. And they have been doing this for decades - since before I was even born. This project is my journey, my exploration, into the "Land that Time Forgot," where I dive into 1970s COBOL logic and scale it to work with 2026 cloud tools.

## Why Forensics?
Mainframes are old but my investigation of them has revealed an impressive level of precision in the way they operate. While modern apps are "move fast and break things," mainframes must do x and calculate y trillions of times and never make a mistake or crash, or else the result is financial doomsday. 

By experimenting with adding forensic auditing to cobol fixed-length 42-byte ledgers using SHA-256 hashing, I hope to achieve a result that ensures when we move data from a 50-year-old "Cylinder" to a modern SSD, there is no data corruption and not a single bit is lost. Not to mention that the SHA-256 adds a level of verifiable tamper-proof security to every peocessed transaction which makes the overall system even more secure and trustworthy.

## What’s Inside?

* **Legacy Logic**: COBOL source code that handles transaction auditing with precision and brevity.

* **Virtual Geometry**: JCL (Job Control Language) templates that talk to the hardware. Here I learn to talk to the hardware: to carve out space on "Virtual 3390" drives using Tracks and Cylinders.

* **The 2026 Bridge**: Python scripts that syntactically beautify these legacy modules, turning EBCDIC data into clean JSON-type code for the modern web.

COBOL is a different world entirely from the one I'm used to. Let's see how far this rabbit hole goes.
